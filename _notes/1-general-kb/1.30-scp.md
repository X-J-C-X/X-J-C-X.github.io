# 1.30-scp

## Overview

## Basic Syntax

Use `scp [options] source destination` to copy files or directories. Source and destination can be local paths or remote `user@host:path` formats. Always specify absolute paths for clarity on remotes

## Details

## Uploading Files

Copy from local to remote.

- Single file: 
  `scp localfile.txt user@remotehost:/remote/path/`​
- Multiple files: 
  `scp file1.txt file2.txt user@remotehost:/remote/path/`
- Entire directory: 
  `scp -r localdir/ user@remotehost:/remote/path/`

## Downloading Files

Copy from remote to local.

- Single file: 
  `scp user@remotehost:/remote/path/file.txt ./`

- Multiple files: 
  `scp user@remotehost:/remote/path/{file1.txt,file2.txt} ./​

- Entire directory: `scp -r user@remotehost:/remote/path/dir/ ./
    

## Key Options

| Option       | Description                         | Example                                          |
| ------------ | ----------------------------------- | ------------------------------------------------ |
| `-r`         | Recursive copy for directories      | `scp -r dir/ user@host:/path/`                   |
| `-C`         | Compress data to speed up transfers | `scp -C file.txt user@host:/path/`               |
| `-v`         | Verbose mode for debugging          | `scp -v file.txt user@host:/path/`               |
| `-P port`    | Specify SSH port (uppercase P)      | `scp -P 2222 file.txt user@host:/path/`          |
| `-i keyfile` | Use specific private key            | `scp -i ~/.ssh/id_rsa file.txt user@host:/path/` |
| `-l limit`   | Bandwidth limit in Kbit/s           | `scp -l 1000 file.txt user@host:/path/`​         |
| `-p`         | Preserve timestamps and modes       | `scp -p file.txt user@host:/path/`​              |
| `-c cipher`  | Select encryption cipher            | `scp -c aes256-ctr file.txt user@host:/path/`    |

## Advanced Use Cases

- Between two remotes (via local host): `scp -3 user1@host1:/path/file.txt user2@host2:/path/`[](https://cheat.sh/scp)
    
- Limit progress output: `scp -q file.txt user@host:/path/`[](https://devhints.io/scp)​
    
- Disable host key checks (scripting): `scp -o StrictHostKeyChecking=no file.txt user@host:/path/`[](https://www.fosslinux.com/138791/scp-command-in-linux-syntax-options-and-examples.htm)​
    

Combine options as needed, e.g., `scp -r -C -v localdir/ user@host:/path/`. Test with small files first, especially in honeypot or Docker environments where paths and permissions matter.
